valdr

A configuration based approach to AngularJS form validation
Download Zip View on GitHub

Model centric

No repeated attributes in your input fields. Define the constraints for your models once and reuse them in all forms of your app.

Simple validation messages

Define the template used to render validation messages once and let valdr take care of the rest. There is no need to repeat yourself for every input field. It even integrates nicely with angular-translate!

Keep your rules in sync

Keep your validation rules in sync with your back-end. For example with the Bean Validation plugin for Java back-ends or by writing your own REST service which provides the validation rules.

Why valdr?

Using the built-in AngularJS validation, a single input field of your form may look something like this:

<div>
    <label for="lastName">Last Name</label>

    <input type="text"
           id="lastName"
           name="lastName"
           ng-model="person.lastName"
           ng-minlength="3"
           ng-maxlength="20"
           required>

    <div class="error" ng-show="myForm.lastName.$dirty && myForm.lastName.$invalid">
        <small class="error" ng-show="myForm.lastName.$error.required">
            Last name is required.
        </small>
        <small class="error" ng-show="myForm.lastName.$error.minlength">
            Last name is required to be at least 3 characters
        </small>
        <small class="error" ng-show="myForm.lastName.$error.maxlength">
            Last name cannot be longer than 20 characters
        </small>
    </div>
</div>

This may be fine for smaller applications... but once your app grows, you end up managing hundreds of validation attributes and messages on dozens of input fields.

Using valdr, you define the constraints and (optionally) the validation messages once:

myApp.config(function(valdrProvider) {
  valdrProvider.addConstraints({
    'Person': {
      'lastName': {
        'Size': {
          'min': 3,
          'max': 20,
          'message': 'Last name is required to be between 3 and 20 characters.'
        },
        'Required': {
          'message': 'Last name is required.'
        }
      }
  });
});

And then use the valdr-type directive on a single or around multiple input fields to apply the validation rules for the given type:

<div valdr-type="Person">
    <label for="firstName">First Name</label>

    <input type="text"
           id="firstName"
           name="firstName"
           ng-model="person.firstName">
</div>

valdr

A model centric approach to AngularJS form validation
Download Zip View on GitHub

Model centric

With valdr, there is no need to repeat the validation attributes in your input fields. Define the constraints for your models once and reuse them in all forms of your app.

Keep your rules in sync

Keep your validation rules in sync with your back-end. Either by using the Bean Validation plugin for Java back-ends or by writing your own REST service which provides the validation rules.

Simple error messages

Define the template used to show error messages once and let valdr take care of the rest. There is no need to repeat yourself for every input field. Need i18n? valdr integrates nicely with angular-translate!

Why valdr?

In your typical AngularJS app, an input field with error messages may look something like this:

<div>
    <label for="lastName">Last Name</label>

    <input type="text"
           id="lastName"
           name="lastName"
           ng-model="person.lastName"
           ng-minlength="3"
           ng-maxlength="20"
           required>

    <div class="error" ng-show="myForm.lastName.$dirty && myForm.lastName.$invalid">
        <small class="error" ng-show="myForm.lastName.$error.required">
            Last name is required.
        </small>
        <small class="error" ng-show="myForm.lastName.$error.minlength">
            Last name is required to be at least 3 characters
        </small>
        <small class="error" ng-show="myForm.lastName.$error.maxlength">
            Last name cannot be longer than 20 characters
        </small>
    </div>
</div>

This may be fine for smaller applications, but once your app grows, you end up managing hundreds of validation attributes and messages for dozens of input fields.

Using valdr, you define the constraints and the messages once...

myApp.config(function(valdrProvider) {
  valdrProvider.addConstraints({
    'Person': {
      'lastName': {
        'Size': {
          'min': 3,
          'max': 20,
          'message': 'Last name is required to be between 3 and 20 characters.'
        },
        'Required': {
          'message': 'Last name is required.'
        }
      }
  });
});

...and use the valdr-type directive to apply the validation rules:

<div valdr-type="Person">
    <label for="firstName">First Name</label>

    <input type="text"
           id="firstName"
           name="firstName"
           ng-model="person.firstName">
</div>
